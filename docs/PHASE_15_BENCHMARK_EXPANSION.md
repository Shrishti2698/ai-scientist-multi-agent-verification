# Phase 15: Benchmark Expansion

## Goal

Strengthen the evaluation setup so the thesis is supported by a broader and more informative benchmark.

## What changed

The benchmark now includes:

- more cases,
- category labels,
- difficulty labels,
- and source notes for traceability.

## Why this matters

A larger benchmark makes it easier to show:

- where the system works,
- where it fails,
- and which components help on which claim types.

## Current categories

- retrieval
- verification
- multi_agent
- critic
- extraction
- generalization

## Research benefit

This makes the results easier to discuss in a thesis chapter because you can analyze outcomes by claim family rather than only with one overall accuracy score.
