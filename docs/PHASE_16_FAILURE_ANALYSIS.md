# Phase 16: Failure Analysis

## Goal

Generate thesis-usable analysis of where each system fails, not just whether it succeeds overall.

## What this adds

The project now exports a failure-analysis report that highlights:

- misclassified benchmark cases,
- category-level error concentrations,
- and common verdict confusions.

## Why this matters

This is the kind of analysis that helps answer panel questions such as:

- where does the multi-agent system still break down?
- does it fail more on contradiction handling or generalization?
- are the remaining errors due to retrieval, verification, or claim formulation?

## Research value

A project becomes much more defensible when it explains its own failures clearly.
